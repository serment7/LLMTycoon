  접근성 5축(키보드/대비/ARIA/스크린리더 라벨/축소 모션) 중 **어떤 것도 떨어뜨리지 않아야** 승인
  대상이 된다.

  "`useReducedMotion` 가드" · "role/aria-live 경로" 를 합격 기준으로 삼는 원칙.

선별 규약:
1. **제거·통합 후에도 회귀 테스트가 전부 녹색** 이어야 한다. 기존 테스트가 잡지 못하는 구성은
   "선제 테스트 추가 후 단순화" 로 단계를 나눈다.
2. **접근성 저하 금지** — 아이콘만 있던 버튼의 텍스트 라벨 제거 같은 축소는 배제한다.
3. **불확실한 장식 효과 제거 우선** — `transition-all`, `hover:-translate-y-0.5` 처럼 맥락 없는
   미세 애니메이션이 최우선 정리 대상.
4. **중복 라벨 우선 통합** — `title` + `aria-label` + 본문 텍스트가 같은 문장을 세 번 말하는 경우
   하나를 남기고 둘을 제거.
5. **정보 압축은 금지** — "본문을 축약해 더 많이 보여주자" 류는 **UX 저해가 아니라 정보 손실**.
